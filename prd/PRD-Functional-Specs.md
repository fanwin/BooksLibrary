## 2. 认证与授权模块 (AUTH)

> **模块标识**: `M01` | **优先级**: P0 | **负责人**: 后端团队  

### 2.1 用户注册 `AUTH-001`

| 属性 | 描述 |
|------|------|
| **用例描述** | 新用户通过填写表单创建图书馆账户 |
| **参与者** | 👤 任意访客 |
| **前置条件** | 无 |
| **基本流程** | 1. 访问注册页 → 2. 填写用户名/邮箱/密码 → 3. 前端校验密码规则 → 4. 提交注册 → 5. 系统创建读者角色账号 |
| **校验规则** | - 用户名 3-50字符，唯一<br>- 邮箱格式有效，唯一<br>- 密码：大小写+数字+特殊字符，≥8位<br>- 注册固定为 reader 角色（防权限提升） |
| **后置处理** | 写入审计日志 `user:create` |

### 2.2 用户登录 `AUTH-002`

| 属性 | 描述 |
|------|------|
| **用例描述** | 已注册用户通过凭证获取访问令牌（需图形验证码防刷） |
| **参与者** | 👤 所有已注册用户 |
| **前置条件** | 账户存在且状态为 active；已获取有效验证码 |
| **基本流程** | 1. 获取验证码图片 → 2. 输入用户名/邮箱 + 密码 + 验证码 → 3. 验证码校验 → 4. bcrypt密码验证 → 5. 生成双令牌(JWT) → 6. 返回令牌对 |
| **扩展能力** | 支持用户名或邮箱两种方式登录；验证码可点击刷新 |
| **异常处理** | ❌ 验证码错误/过期 → 400（自动刷新验证码）<br>❌ 账户不存在 → 401<br>❌ 密码错误 → 401<br>❌ 账户禁用 → 403 (提示状态) |
| **令牌策略** | Access Token: **1小时（会话生命周期）** / Refresh Token: 7天 |
| **会话约束** | ⭐ 登录后会话有效期 **1 小时**；前端每 30 秒检测；距过期 5 分钟弹窗提醒；过期自动登出 |
| **安全机制** | ⭐ 图形验证码：4位字符、5分钟有效期、一次性使用、大小写不敏感 |

### 2.2.1 登录验证码 `AUTH-002a` ⭐ 新增

| 属性 | 描述 |
|------|------|
| **功能名称** | 登录图形验证码 |
| **优先级** | P0 - 必须 |
| **状态** | ✅ 已实现 |
| **目的** | 防止暴力破解/自动化脚本攻击登录接口 |

**验证码规则**:
| 规则项 | 值 |
|--------|-----|
| 字符集 | 大写字母(A-Z, 排除I/O) + 数字(2-9)，共32个可用字符 |
| 字符长度 | 固定 4 位 |
| 有效期 | 300 秒（5 分钟） |
| 使用次数 | 一次性（校验成功后立即失效，不可重复使用） |
| 大小写 | 不敏感（统一转为大写比较） |
| 图片格式 | PNG (Base64 编码) |
| 图片尺寸 | 120 × 46 px |
| 干扰元素 | 随机浅色背景 + 3~5条干扰线 + 50~100个干扰点 |
| 存储方式 | 服务端内存字典（生产环境建议替换为 Redis） |
| 清理策略 | 每次生成时清理过期记录 |

**前端交互规范**:
```
┌──────────────────────────────────────────────┐
│  [🔒 密码输入框          ]                    │
│                                              │
│  [A7XK        ] [🖼️验证码图片]   ← 点击刷新  │
│              看不清？换一个                    │
│                                              │
│  [           登 录           ]                │
└──────────────────────────────────────────────┘

交互规则：
1. 页面加载时自动请求验证码图片
2. 用户点击图片或"换一个"文字可刷新验证码
3. 刷新后清空已输入的验证码内容
4. 登录失败后自动刷新验证码（防止暴力重试）
5. 验证码输入框最大长度 4 位，字母间距加宽
```

## 4. 借阅流通模块 (BORROWS)

> **模块标识**: `M03` | **优先级**: P0 | **负责人**: 流通管理员  

### 4.1 借阅资格检查 `BORROW-001`

| 属性 | 描述 |
|------|------|
| **用例描述** | 在借书前全面检查读者的借阅资格 |
| **检查项清单** | □ 账户状态是否 active<br>□ 未达最大借阅数(默认10本)<br>□ 无逾期未还图书<br>□ 累计未缴罚款未达冻结阈值(默认50元) |
| **返回结果** | eligible: bool + reason: string(不通过原因) |

### 4.2 单本借阅 `BORROW-002`

| 属性 | 描述 |
|------|------|
| **用例描述** | 为读者办理单本图书借阅手续 |
| **权限要求** | circulation_admin 或 super_admin |
| **业务规则** | ① 通过资格检查 → ② 副本状态必须为 available<br>③ 无其他读者的有效预约(非本人) → ④ 如本人有 READY 状态预约则自动履约<br>⑤ 应还日期 = 工作日计算(跳过周末和法定节假日) |
| **联动更新** | 副本状态→borrowed, 图书available_copies-1, 可能触发图书status变化 |

### 4.3 批量借阅 `BORROW-003`

| 属性 | 描述 |
|------|------|
| **用例描述** | 一次扫码多本图书完成借阅（提升前台效率） |
| **输入** | user_id + copy_ids[] |
| **输出** | success_items[] + fail_items[](含失败原因) |
| **边界控制** | 每处理一本后实时检查是否已达借阅上限 |

### 4.4 归还图书 `BORROW-004`

| 属性 | 描述 |
|------|------|
| **用例描述** | 处理图书归还，含异地还书支持 |
| **参与者** | 👤 管理员可代还 / 读者可自还自己的 |
| **核心逻辑** | ① 计算逾期天数和罚款金额<br>② 免罚期内(grace_days=3)不收费<br>③ 之后按日罚款(daily_fine=0.5元/天)<br>④ 支持记录归还分馆(return_branch) |
| **归还后处理** | → 检查是否有等待预约者<br>　　有 → 副本改为reserved, 通知首位排队者<br>　　无 → 副本改为available, 恢复可借数 |
| **冻结机制** | 累计罚款 ≥ 冻结阈值 → 自动冻结(suspended)读者借阅权限 |

### 4.5 续借 `BORROW-005`

| 属性 | 描述 |
|------|------|
| **用例描述** | 读者在线延长借阅期限 |
| **续借条件** | ① 借阅状态须为 active<br>② 未超过应还日期(逾期不可续)<br>③ 未达最大续借次数(默认2次)<br>④ 该书未被他人预约 |
| **续借时长** | 默认延长15个工作日（考虑节假日顺延） |

### 4.6 借阅记录查询 `BORROW-006`

| 属性 | 描述 |
|------|------|
| **用例描述** | 查询借阅历史记录列表 |
| **权限差异** | 管理员：可查所有人，支持按用户/状态/关键词筛选<br>读者：只能查自己的记录 |
| **展示信息** | 书名/条码/借阅日期/应还日/归还日/状态/罚款/续借次数/是否逾期/逾期天数 |
| **特殊标注** | 逾期记录以红色高亮显示 |

### 4.7 逾期信息查询 `BORROW-007`

| 属性 | 描述 |
|------|------|
| **用例描述** | 获取某条借阅的详细逾期信息 |
| **返回内容** | 是否逾期 / 应还日 / 当前日 / 总逾期天数 / 免罚天数 / 应缴费天数 / 日罚款率 / 预估罚款金额 |

## 9. 系统管理模块 (SYSTEM)

### 9.1 用户管理 `SYS-001`
- **CRUD**: 创建/查询/更新/删除用户
- **搜索**: 用户名/邮箱/手机号/读者证号模糊搜索
- **筛选**: 按角色/状态筛选
- **批量操作**: 禁用/激活用户
- **读者证**: 办理/挂失/补换（自动生成证号：RD+年+类型码+序号）
- **借阅参数联动**: 读者类型(student/staff/public/admin)自动匹配不同的最大借阅数和借阅天数

### 9.2 角色权限 `SYS-002`
- **动态角色**: 超级管理员可创建自定义角色
- **权限分配**: 30+细粒度权限点的组合
- **内置保护**: 5个系统角色不可删除
- **初始化**: 一键将枚举角色同步到数据库 RBAC 表

### 9.3 分类管理 `SYS-003`
- **树形结构**: 支持无限层级分类
- **操作**: 增删改查
- **删除保护**: 有子分类或有图书引用时不可删除

### 9.4 系统配置 `SYS-004`
- **键值对配置**: 借阅天数/上限/罚款费率/阈值等
- **权限**: 仅超级管理员可修改

### 9.5 节假日管理 `SYS-005`
- **作用**: 影响应还日期的计算（跳过节假日）
- **批量导入**: 支持一次性导入全年节假日
- **去重**: 同一天不可重复创建

### 9.6 消息通知 `SYS-006`
- **通知类型**: system(系统) / reservation_ready(预约到书) / fine(罚款相关)
- **操作**: 标记已读 / 全部已读 / 删除
- **铃铛组件**: 顶部导航栏实时显示未读数

### 9.7 审计日志 `SYS-007`
- **不可篡改**: 只增不改的设计原则
- **记录内容**: 操作人/操作类型/目标对象/IP/时间<br>+ **操作前数据快照**(JSON)<br>+ **操作后数据快照**(JSON)
- **筛选**: 按类型/操作人/时间范围/关键词
- **导出**: CSV格式下载

### 9.8 荐购管理 `SYS-008`
- **流程**: 读者提交荐购(书名/作者/ISBN/理由) → 管理员审核(批准/拒绝/已采购)
- **权限**: 管理员可查看所有；读者只能看自己的
